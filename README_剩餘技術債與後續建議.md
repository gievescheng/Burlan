# 柏連專案剩餘技術債與後續建議

## 目前已經做好的
- 柏連與潔沛的執行期私有資料已分開
- 正式文件、稽核計畫、不符合、校正、設備、供應商、KPI 已有主線功能
- 後端已拆出文件、作業、整合三組處理器
- 前端已拆出文件、稽核、作業、設備供應商、KPI 幾個模組
- 測試目前可穩定通過

## 還沒做完，但建議後續處理的重點

### 1. Git 邊界仍在上層 `Codex`
- 目前柏連專案工作區已清楚，但 Git 真正根目錄仍在：
  `C:\Users\USER\Documents\Codex`
- 這表示版本管理和柏連工作區邊界還不是同一層。
- 影響：
  日後如果直接在上層看 Git，還是會混到別的東西。

### 2. 前端還有部分頁面沒完全拆開
- `訓練管理`
- `環境監測`
- `生產 / 品質紀錄`
- `通知提醒`
- `AI 工作台`
- `記錄匯出`
- 這些還在主整合檔或較大模組裡。

### 3. `server.py` 雖然已經變薄，但還可再整理
- 目前路由入口還在主檔
- 後續可以考慮把不同模組的 route 註冊再集中整理
- 但這一步不急，因為現在已比之前穩定很多

### 4. 工作區檢查腳本的 PowerShell 中文顯示
- 檢查結果本身正常
- 但 PowerShell 視窗顯示中文路徑時仍可能亂碼
- 這比較像終端顯示問題，不是資料檔真的壞掉

### 5. 中文文件仍需要定期體檢
- 雖然目前已補 UTF-8 檢查
- 但 Excel、Markdown、說明文件只要來源混亂，還是有機會再出現亂碼
- 建議重大整理後固定跑一次檢查腳本

## 我建議的後續順序

### 第一優先
- 把前端剩下幾個大頁面再拆開
- 目標是讓主整合檔再變小一點

### 第二優先
- 決定是否把柏連專案獨立成自己的 Git 邊界
- 這會讓之後版本管理更乾淨

### 第三優先
- 補更多回歸測試
- 尤其是：
  `1.` 文件預覽
  `2.` 稽核附件對應
  `3.` 不符合匯入與修正
  `4.` 校正與設備日期重算

## 目前不建議做的
- 不建議大重寫
- 不建議一次把前後端全部換框架
- 不建議為了漂亮架構去打掉現有可用流程

## 一句話結論
- 柏連專案目前已從「功能堆在一起」走到「主線可用、結構逐步清楚」
- 接下來重點不是重做，而是持續小步整理，讓它更穩、更好維護
