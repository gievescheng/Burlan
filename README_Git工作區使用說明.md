# 柏連專案 Git 工作區使用說明

## 你現在要記住的兩個位置
- 柏連原始文件根目錄：
  `C:\Users\USER\Documents\Codex\ISO 9001 SOP-柏連正式版`
- 柏連系統程式工作區：
  `C:\Users\USER\Documents\Codex\ISO 9001 SOP-柏連正式版\_柏連稽核系統`

## 為什麼還會看到上層 `Codex`
- 目前真正的 Git 根目錄仍在：
  `C:\Users\USER\Documents\Codex`
- 所以如果直接在上層跑 `git status`，會把柏連以外的東西也一起算進去。
- 這不代表柏連系統壞掉，只是版本管理邊界還在更上層。

## 日常建議怎麼看專案狀態
- 直接用：
  `檢查柏連專案工作區.bat`
- 這個入口會幫你檢查：
  `1.` 柏連專案內有沒有跑進私有執行資料
  `2.` 文字檔是不是 UTF-8
  `3.` 有沒有殘留 `JEPE` 或 `潔沛`
  `4.` 只看柏連專案範圍的 Git 狀態

## 你之後看到 `?? ./` 是什麼意思
- 這通常代表 Git 是從更上層看柏連整個資料夾。
- 白話講，就是 Git 知道「這裡有一整包柏連專案」，但它的根還不在柏連這一層。
- 目前這不影響柏連系統使用，只是提醒你：
  Git 邊界和柏連工作區邊界還不是同一層。

## 柏連和潔沛現在怎麼分開
- 柏連執行期私有資料固定使用：
  `%APPDATA%\AutoAudit\burlan_qms`
- 這表示柏連的系統紀錄、補充資料、暫存資料，不再和潔沛共用同一份私有資料夾。
- 共用的是架構，不是執行資料。

## 目前最安全的使用方式
- 開柏連系統時，從這裡進：
  `C:\Users\USER\Documents\Codex\ISO 9001 SOP-柏連正式版\_柏連稽核系統\啟動柏連稽核系統.bat`
- 看柏連專案狀態時，從這裡進：
  `C:\Users\USER\Documents\Codex\ISO 9001 SOP-柏連正式版\_柏連稽核系統\檢查柏連專案工作區.bat`

## 後續如果還要再整理
- 如果之後要把 Git 邊界也完全切開，就要另外把柏連專案獨立成自己的版本庫。
- 這一步目前還沒做，但已經先把日常檢查和執行資料切開了。
