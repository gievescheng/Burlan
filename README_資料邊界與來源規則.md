# 柏連專案資料邊界與來源規則

## 1. 專案內有四種資料
- 柏連原始受控文件：
  `C:\Users\USER\Documents\Codex\ISO 9001 SOP-柏連正式版`
- 柏連系統程式碼：
  `C:\Users\USER\Documents\Codex\ISO 9001 SOP-柏連正式版\_柏連稽核系統`
- 系統輸出檔：
  例如 `index.html`、匯出報表、整理後 Excel
- 執行期私有資料：
  `%APPDATA%\AutoAudit\burlan_qms`

## 2. 最重要的原則
- 原始受控文件優先。
- 系統補充紀錄只做補充，不反蓋原始正式資料。
- 遇到版本衝突、缺欄位或找不到對應時，標記待確認，不自動亂猜。
- 柏連不得讀入潔沛資料。

## 3. 各模組主要資料來源
- 正式文件：
  `柏連正式文件主清單.xlsx`
- 稽核計畫：
  `17內部稽核管理程序\記錄`
- 不符合管理：
  `20不符合及矯正措施管理程序\記錄`
- 校正管理：
  `9量測資源管理程序\記錄\9.1量規儀器一覽表`
  和
  `9量測資源管理程序\記錄\9.2量規儀器履歷表`
- 設備保養：
  `8設施設備管理程序\記錄`
- 供應商管理：
  `12採購及供應商管理程序\記錄`
  和
  `三階文件\供應商滿意度清單`
- KPI：
  `3目標管理程序\記錄\3.1品質目標管制表`

## 4. 系統補充紀錄現在怎麼用
- 校正更新：
  先存系統紀錄，不直接改原始 Word 或 Excel
- 設備保養：
  保養歷程存在系統資料，用來追蹤最近保養與下次保養
- 訓練管理：
  先以系統資料保存人員與訓練紀錄
- 不符合修正：
  可在系統內補日期、責任人、改善措施等欄位

## 5. 哪些資料不該混進版本管理
- Office 暫存檔
- 鎖定檔
- 執行快取
- 中間輸出檔
- 私有執行資料

## 6. 如何確認目前邊界正常
- 跑：
  `檢查柏連專案工作區.bat`
- 它會檢查：
  `1.` 柏連專案內沒有私有執行資料
  `2.` 掃描到的文字檔都是 UTF-8
  `3.` 沒有殘留 `JEPE` 或 `潔沛`

## 7. 這份規則未來怎麼用
- 新功能要接資料前，先決定：
  `1.` 正式來源是哪一份文件
  `2.` 系統補充資料要不要存在
  `3.` 哪些欄位可人工修正
  `4.` 哪些情況只能標記待確認
- 只要照這四個問題走，就比較不會再把資料混亂。
